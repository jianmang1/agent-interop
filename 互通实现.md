# 笔记本 ⇄ 工位电脑 跨设备互通实现方案

## 背景

笔记本（192.168.3.x）与工位电脑（192.168.1.x）不在同一子网，无法直接网络互通。经过多次尝试（飞书机器人直连、Webhook 直连、飞书群聊桥梁），最终采用 **GitHub 仓库作为消息队列** 实现异步文件交换。

## 实现历程

### 尝试过的方案

| 方案 | 结果 | 原因 |
|------|------|------|
| 飞书群聊发截图 | ✅ 可用但需手动下载 | 同一飞书账号跨设备登录 |
| Webhook 直连 | ❌ 不通 | 两台机器在不同网段（192.168.3.x vs 192.168.1.x） |
| Hermes pairing | ❌ 不适用 | pairing 是用户授权机制，非设备互连 |
| **GitHub 队列** | ✅ **最终方案** | 异步可靠，双方都能 push/pull |

## 最终方案：GitHub 队列协作协议

### 仓库信息

- **URL**: https://github.com/jianmang1/agent-interop
- **本地路径（笔记本）**: `D:\agent互通\`
- **本地路径（工位电脑）**: `E:\agent互通\`

### 目录结构

```
agent-interop/
├── requests/                  # 请求队列
│   ├── README.md
│   ├── request-{ts}-{rand}.json   # 请求文件（发起方写）
│   └── processed_request-*        # 已处理的请求（处理方重命名）
├── results/                   # 结果队列
│   ├── README.md
│   ├── result-{ts}-{rand}.json    # 结果文件（处理方写）
│   └── processed_result-*         # 已处理的结果（发起方重命名）
| `poll-notebook.ps1`          # 笔记本端轮询脚本
| `poll-office.ps1`            # 工位电脑端轮询脚本（含截图/命令/文件/状态四种请求处理）
| `skills/github-queue-interop.md`  # Hermes skill 定义（笔记本端加载）
| `skills/human-layer-framework.md` # 人类多层分析框架 skill
├── desktop_*.png              # 桌面截图文件
├── 互通实现.md                 # 本文档
└── README.md                  # 仓库说明
```

### 请求/响应格式

**请求**（`requests/request-{时间戳}-{随机4位}.json`）：

```json
{
  "type": "screenshot",
  "msg": "请截一张工位电脑的桌面截图",
  "reply_to": "results/result-{时间戳}-{随机4位}.json",
  "timestamp": "2026-07-15 11:52:44"
}
```

**响应**（`results/result-{时间戳}-{随机4位}.json`）：

```json
{
  "type": "screenshot",
  "status": "completed",
  "result_file": "desktop_screenshot.png",
  "timestamp": "2026-07-15 11:52:44",
  "description": "工位电脑桌面截图",
  "note": "如需更多信息请继续提需求"
}
```

### 工作流程

#### 笔记本 → 工位电脑

```
1. 笔记本写 requests/request-*.json
2. git add / commit / push
3. 工位电脑轮询 → git pull → 发现新请求
4. 工位电脑执行任务（截图/处理）
5. 工位电脑写 results/result-*.json + 生成附件文件
6. 工位电脑 git add / commit / push
7. 笔记本轮询 → git pull → 收到结果
8. 笔记本重命名 result-* → processed_result-* 并 push
```

#### 工位电脑 → 笔记本（反向同理）

```
1. 工位电脑写 requests/request-*.json
2. git push
3. 笔记本轮询 → 处理 → 写结果 → push
```

### 轮询脚本
